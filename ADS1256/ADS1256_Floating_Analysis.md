# ADS1256 悬空输入分析与工程处理

结论：软件可以把 ADS1256 初始化、同步、校准、滤波、单端/差分读数封装得更易用，但不能把物理悬空的模拟输入变成可靠电平。正在测量的 AINx 必须有明确电气状态。

## 为什么悬空会乱

数据手册里的关键事实：

- 单端测量是 `AINx - AINCOM`，不是天然的 `AINx - 树莓派 GND`。
- 缓冲关闭时，输入端进入开关电容采样结构，内部电容会持续充放电；高阻或悬空节点会被采样动作、寄生电容、漏电、板载前端网络影响。
- 缓冲开启可以提高输入阻抗，但输入对地范围被限制为 `AGND ~ AVDD - 2.0V`。本模块如果只能 3.3V 供电，buffer 开启后上限约 1.3V，不适合 0~3.3V 输入。
- SDCS 传感器检测电流源会向输入端注入 0.5uA/2uA/10uA，必须默认关闭。

所以，`AIN0` 悬空时看到 0.6V、1.4V 或缓慢漂移，都不是 ADC “坏了”，而是输入节点没有被定义。

## 软件能做什么

驱动层应做到：

- 强制关闭 `ADCON.SDCS`，避免传感器检测电流源抬高输入。
- 0~3.3V 输入默认关闭 buffer。
- 修改 MUX/PGA/DRATE/buffer 后执行 `SYNC/WAKEUP`，等待 `DRDY` 再读。
- 上电或配置变化后执行自校准。
- 读数前丢弃若干样本，再用 median/mean 做稳定读数。
- 提供单端和差分的简洁 API。

当前 `ads1256_bitbang.py` 已补充：

```python
adc.configure_single_ended(channel=0, buffer_enabled=False)
voltage = adc.read_stable_voltage(samples=8, discard=3, method="median")

adc.configure_differential(positive_channel=0, negative_channel=1)
voltage = adc.read_stable_voltage(samples=8, discard=3)
```

软件不能做到：

- 自动判断一个悬空节点的真实电压。
- 用校准抵消输入端没有参考的问题。
- 在没有硬件泄放路径时保证“接上信号立刻准确”。

## 硬件必须做什么

### 通用电压输入

推荐每个使用的 AIN 通道都加输入前端：

```text
信号源输出 -- 100Ω~1kΩ -- AINx
AINx -- 1nF~100nF -- COM/GND
AINx -- 47kΩ~220kΩ -- COM/GND
信号源 GND -- COM/GND
```

说明：

- 串联电阻限制瞬态电流，并和电容形成低通。
- 电容滤除高频噪声，靠近 ADC 输入端。
- 下拉/泄放电阻让未连接时有确定状态。
- 具体电阻值需要按信号源输出阻抗和分压误差折中。

### 按钮 ADC 分压模块

按钮模块本身就是分压输出。外加下拉会改变各按键电压，所以做法是：

1. 先选一个下拉值，如 `220kΩ` 或 `100kΩ`。
2. 重新记录 `NONE/UP/DOWN/LEFT/RIGHT/A/B/C/D/UNPLUGGED` 电压。
3. 更新 `BUTTON_LEVELS`。

如果 `100kΩ` 仍然无法让未连接状态稳定，说明该模块/板载输入网络不适合用 ADS1256 读按钮。按钮不需要 24 位 ADC，优先改成 GPIO 矩阵键盘、ADS1115、MCP3008 或专用按键 ADC。

### 光电流跨阻输出

光电流测量建议不要让传感器直接面对 ADS1256 输入：

```text
光电二极管 -> 跨阻放大器 -> 低阻电压输出 -> RC输入网络 -> ADS1256
```

跨阻放大器输出是低阻源，能驱动 ADS1256 的开关电容输入，这才是 ADS1256 擅长的工作方式。

## 最终建议

- ADS1256 留给光电流跨阻电压这种真正需要高分辨率的信号。
- 按钮模块如果继续用 ADS1256，必须加泄放电阻并重新标定。
- 不要期待软件消除悬空。软件只能识别、滤波、丢弃过渡样本；硬件必须定义输入节点。
